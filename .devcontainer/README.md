# Dev Container Configuration

Эта конфигурация настраивает среду разработки на базе **Ubuntu 22.04** вместо Alpine Linux, что решает проблему совместимости Prisma.

## Что включает конфигурация

- **Ubuntu 22.04** - стабильный дистрибутив Linux с полной поддержкой Prisma
- **Node.js 20** - последняя LTS версия
- **PostgreSQL 14** - база данных
- **Автоматическая настройка** - все зависимости устанавливаются автоматически

## Как использовать

### В GitHub Codespaces

1. **Пересоздайте Codespace:**
   - Откройте меню Codespaces (три точки)
   - Выберите "Rebuild Container" или создайте новый Codespace
   - Codespace автоматически использует новую конфигурацию

2. **После пересоздания:**
   - Все зависимости будут установлены автоматически
   - База данных будет настроена
   - Проект будет готов к работе

### Локально (с VS Code)

1. Установите расширение "Dev Containers" в VS Code
2. Откройте проект в VS Code
3. Нажмите `F1` → "Dev Containers: Reopen in Container"
4. Дождитесь сборки контейнера

## Порты

- **3000** - Frontend (Vite)
- **3001** - Backend API (NestJS)
- **5432** - PostgreSQL

## Команды для запуска

После пересоздания Codespace:

```bash
# Создать администратора
cd crm-backend
npm run create:admin

# Запустить бэкенд (в одном терминале)
npm run dev

# Запустить фронтенд (в другом терминале)
cd ../CRM
npm run dev
```

После пересоздания Codespace все файлы проекта сохранятся, но нужно будет:
