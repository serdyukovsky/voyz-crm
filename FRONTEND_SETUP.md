# Инструкция по запуску фронтенда локально

## Бэкенд уже запущен на сервере

Бэкенд работает на сервере по адресу: **http://91.210.106.218:3001**

## Шаги для запуска фронтенда локально

### 1. Клонируйте репозиторий (если еще не сделали)

```bash
git clone https://github.com/pgbxxms8wg-max/voyz-crm.git
cd voyz-crm/CRM
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Создайте файл `.env.local`

Создайте файл `.env.local` в директории `CRM/` со следующим содержимым:

```env
# Backend API Base URL - указываем IP сервера
VITE_API_URL=http://91.210.106.218:3001/api

# WebSocket URL для real-time обновлений
VITE_WS_URL=http://91.210.106.218:3001/realtime
```

### 4. Запустите фронтенд

```bash
npm run dev
```

Фронтенд будет доступен по адресу: **http://localhost:3000**

### 5. Войдите в систему

Откройте браузер и перейдите на `http://localhost:3000/login`

**Учетные данные администратора:**
- **Email:** `admin@example.com`
- **Password:** `admin123`

## Проверка подключения

Если фронтенд не может подключиться к бэкенду, проверьте:

1. Бэкенд запущен на сервере:
   ```bash
   curl http://91.210.106.218:3001/api/health
   ```
   Должен вернуть: `{"status":"ok","timestamp":"..."}`

2. Файрволл на сервере разрешает подключения на порт 3001

3. Переменные окружения в `.env.local` указаны правильно

## Дополнительная информация

- **Swagger документация API:** http://91.210.106.218:3001/api/docs
- **Health check:** http://91.210.106.218:3001/api/health
- **База данных:** PostgreSQL в Docker контейнере `crm-postgres`

Для просмотра логов бэкенда на сервере:
