# Инструкция по запуску бэкенда

## Проблема: Бэкенд не отвечает

Если бэкенд не отвечает на запросы, выполните следующие шаги:

### 1. Остановите все процессы NestJS

```bash
pkill -f "nest start"
```

### 2. Убедитесь, что база данных доступна

Проверьте подключение к базе данных:

```bash
cd crm-backend
npm run check:admin
```

Если база данных не доступна, вам нужно:
- Установить и запустить PostgreSQL
- Или использовать Docker для запуска PostgreSQL

### 3. Запустите бэкенд в отдельном терминале

Откройте новый терминал и выполните:

```bash
cd crm-backend
npm run start:dev
```

Дождитесь сообщения:
```
Application is running on: http://localhost:3001
```

### 4. Проверьте, что бэкенд работает

В другом терминале:

```bash
curl http://localhost:3001/api/docs
```

Должна открыться страница Swagger.

### 5. Проверьте логи

Если бэкенд не запускается, проверьте логи на наличие ошибок:
- Ошибки подключения к базе данных
- Ошибки валидации конфигурации
- Ошибки зависимостей

### Альтернатива: Запуск через Docker

Если у вас установлен Docker, можно запустить все через docker-compose:

```bash
cd crm-backend
docker compose up
```

Это запустит и PostgreSQL, и бэкенд.
